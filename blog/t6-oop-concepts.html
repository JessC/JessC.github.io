Object Oriented Programming Concepts - Ruby Variables

Jan 31, 2015

When first getting started in ruby, you might just start out writing one or two lines of code. Maybe something that will complete a simple math equation. After that you're likely to start defining methods. At this point you'll be getting some experience creating simple variables like:

But before you know it, you'll start creating classes where you'll be defining multiple methods inside. Sometimes those variables need to be access throughout you're class. Lets say I have a class that holds information about people. When I call an instance of that class, I may want to use a persons name throughout multiple methods. I could do that by simple using an '@' in front of the variable. So if I have a variable called 'name' I could set it to be used throughout the instance of that class by doing @name = name. If I were called up multiple instances of a class but wanted the variable to be the same throught all of them I would use '@@'' instead of a single '@'

Eventually you'll be making larger more complicated programs. Now, lets say you want to have a username for the person running that program. In this case you'll want to use a global variable that's visable throughout the programs. To do this, a '$' preceeds the variable. If the program has variables set to fixed data such as profits you write the variable in uppercase to make it a constant. So this would be $global and this would be CONSTANT. The only real difference is that a CONSTANT variable will return a warning if the code tries to change it.

It's important to know the difference between the variables and when to use the different scopes. Keeping variables local is beneficial when you want to keep using generic names like num and x inside your methods. But enlarging the scope to instance, class, or global can help cut down redundancy by being able to define a variable the least amount of times possible.
